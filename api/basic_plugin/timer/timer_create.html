timerCreate¶

Timer を作成するAPIです。

Timer 設定マスタの識別子( configId )をリクエストし、その設定値を使用した Timer を作成し開始します。開始時付与 Content が定義されていた場合は、付与処理を行います。

Timer はプレイヤー毎に複数作成することが可能で、それぞれの Timer を識別するために、作成時にはユニークな statusId をリクエストする必要があります。

作成された Timer は state に RUNNING が入り、カウント状態となります。

timerCreate(TimerCreateRequest) => TimerCreateResult

Requests¶

  • statusId (int)

    作成する Timer の 識別子

  • configId (int)

    使用する Timer マスタ 識別子

Returns¶

  • success (bool)

    APIの成否

  • updateTimer (optional, Timer)

    作成した Timer の状態

  • updateInventoryElementList (optional, list<InventoryElement>)

    更新された Inventory のリスト

Examples¶

リクエストの例¶

{
    statusId: 1,
    configId: 100
}

レスポンスの例¶

{
    success: true,
    updateTimer: {
        statusId: 1,
        configId: 100,
        state: 'RUNNING'
        point: 0,
        lastUpdate: 1459436400000,  /* Epoch milliseconds : 2016/4/1 0:0:0 */
        countInterval: 100,
        maxPoint: 500
    },
    updateInventoryElementList: [
        {
            inventoryId: 1001,
            contentId: 10,
            contentType: 'type-01',
            amount: 10,
            description: 'Present',
            isPayment: false
        }
    ]
}

Errors¶

存在しない configId や、不正な statusId をリクエストした場合、エラーを返します。

エラーコード 説明
20700 不正な statusId がリクエストされました
20701 不正な configId がリクエストされました
20702 既に statusId が設定された Timer が存在しています
