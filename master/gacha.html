Gacha マスタ¶

Gacha マスタでは、 Gacha を実行した際に排出を行う Content 、抽選時の重み、対価として消費が行われる Content を定義します。

排出を行う Content はパック 識別子 でまとめることができ、複数のパックを定義すると排出 Content を複数にすることができます。

以下にマスタの持つ属性値を記載します。

Gacha マスタの持つ属性値¶

  • data_version

    定義を行うマスタのバージョン

    型 : 文字列

    必須 : yes

    入力可能文字数 : 1-32

    備考 : DataVersion については DataVersion を参照

  • gacha_id

    Gacha の 識別子

    型 : 整数値

    必須 : yes

    範囲 : 0-9999

  • gacha_type

    Gacha の種類

    型 : 文字列

    必須 : yes

    備考 : 定義可能な GachaType は GachaType 一覧 を参照

  • active_from

    Gacha の有効期限の開始日時を示す

    型 : 文字列 (フォーマット “yyyy-MM-dd HH:mm:ss”)

    必須 : no

  • active_to

    Gacha の有効期限の終了日時を示す

    型 : 文字列 (フォーマット “yyyy-MM-dd HH:mm:ss”)

    必須 : no

  • consume_content_id

    対価として消費される Content の 識別子

    型 : 整数値

    必須 : yes

    範囲 : 0-99999

  • consume_amount

    対価として消費される Content の個数

    型 : 整数値

    必須 : yes

    範囲 : 0-99999

  • pack_id

    Gacha で排出される Content をまとめたパック 識別子

    型 : 整数値

    必須 : yes

    範囲 : 0-999

    備考 : 1つのパック識別子でまとめることのできる Content は 99個まで

  • add_content_id

    排出された際に付与される Content の 識別子

    型 : 整数値

    必須 : yes

    範囲 : 0-99999

  • add_amount

    排出された際に付与される Content の個数

    型 : 整数値

    必須 ：yes

    範囲 : 0-99999

  • weight

    抽選の際に使用される重み値

    型 : 整数値

    必須 : yes

    範囲 : 0-99999

plain Gacha の設定例¶

課金コンテンツとして定義した Content を5つ消費して、「キャラクターA」,「キャラクターB」,「キャラクターC」の Content の中からランダムに1つのキャラクター抽選する Gacha の設定例です。 「キャラクターC」だけその他のキャラクターよりも2倍の確率で抽選されるように設定します。
